- 기준: 현재 로컬 MVP(v0.6.0) 이후 우선 작업

## 1. 추천 작업 순서

1. [~] 자기소개 생성 품질 개선
2. [~] 채용공고 구조화와 URL 추출 안정화
3. [~] 테스트 자동화와 회귀 방지 체계 추가
4. [~] 단계별 수정과 재생성 UX 정교화
5. [ ] PDF 입력과 배포형 구조 검토

## 2. 왜 이 순서인가

- 현재 사용자 체감 품질은 여전히 `자기소개 결과`와 `채용공고 구조화/URL 추출 정확도`에 가장 크게 좌우됩니다.
- 코드 기준으로 단계 가드, stale 안내, 진행 모달, 이전 결과 비교 UI는 이미 들어가 있어 UX는 사용 가능한 기준선에 도달했습니다. 남은 UX 작업은 주로 마감도 개선입니다.
- 반면 URL 추출 로직과 소개글 후처리 규칙은 빠르게 복잡해지고 있고, 아직 CI 자동화가 없어 회귀 방지 체계를 한 단계 더 올려야 합니다.

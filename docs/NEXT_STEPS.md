# Next Steps

- 문서 버전: v0.1
- 작성일: 2026-03-06
- 기준: 현재 로컬 MVP 이후 우선 작업

## 1. 추천 작업 순서

1. 자기소개 생성 품질 개선
2. 채용공고 구조화 정확도 개선
3. 단계별 수정/재생성 UX 정교화
4. E2E 테스트와 회귀 방지 체계 추가
5. PDF 입력과 배포형 구조 검토

## 2. 왜 이 순서인가

- 현재 사용자 체감 품질은 `자기소개 결과`와 `채용공고 구조화 품질`에 가장 크게 좌우된다.
- UI 흐름은 이미 MVP 기준으로 usable한 상태이므로, 다음 단계는 생성 정확도와 재생성 신뢰성을 끌어올리는 것이 맞다.
- 그 다음에 테스트/E2E를 보강해야 이후 프롬프트/스키마 수정이 잦아져도 회귀를 빠르게 잡을 수 있다.

## 3. 우선순위별 작업

### P1. 자기소개 생성 품질 개선

목표:
- 소개문을 더 덜 일반적이고, 공고-이력서 매칭 근거가 보이는 결과로 바꾼다.

작업:
- `generate-intro` 출력 스키마 확장 검토
- 추천 후보:
  - `oneLineIntro`
  - `shortIntro`
  - `longIntro`
  - `fitReasons[]`
  - `matchedSkills[]`
  - `missingButRelevant[]`
  - `tone`
- 자기소개 생성 전, `resume`와 `company`의 매칭 포인트를 중간 구조로 먼저 계산하는 단계 추가 검토
- 프롬프트에서 금지할 것 명시:
  - 근거 없는 과장
  - 공고에 없는 표현 임의 추가
  - 지나치게 일반적인 문장 반복

완료 기준:
- 결과가 공고 요구사항 3개 이상과 직접 연결된다.
- 같은 이력서로 회사만 바꿨을 때 결과 차이가 명확히 보인다.

### P2. 채용공고 구조화 정확도 개선

목표:
- 붙여넣은 공고 텍스트가 지저분해도 핵심 필드가 안정적으로 분리되게 만든다.

작업:
- `company-to-json` 스킬에서 추출 규칙 강화
- `requirements`, `preferredSkills`, `techStack`, `jobDescription` 분리 기준 재정의
- 공고에 흔한 노이즈 처리:
  - 복지/문화 문구
  - 중복 bullet
  - 링크/마크다운 잔여 문자열
- 샘플 공고 fixture 10~20개 확보 후 회귀 테스트 추가

완료 기준:
- 동일 공고 재분석 시 핵심 필드 흔들림이 줄어든다.
- `필수 요구사항` 누락률이 눈에 띄게 줄어든다.

### P3. 단계별 UX 정교화

목표:
- 지금의 3단계 흐름을 유지하면서도 수정-재생성 비용을 더 낮춘다.

작업:
- 각 단계에 `마지막 확정 시각` 표시
- 어떤 변경 때문에 `결과 재생성 필요`가 되었는지 원인 메시지 표시
- 결과 페이지에서 `회사 공고만 바뀜` 같은 변경 원인 배지 표시
- 비교 UI를 텍스트 diff 또는 강조 표시 형태로 개선
- 필드 수가 많은 배열 섹션에 접기/펼치기 추가

완료 기준:
- 사용자가 현재 막힌 이유와 다음 행동을 화면만 보고 판단할 수 있다.

### P4. 테스트/운영 안정성

목표:
- 프롬프트, 스키마, 상태 규칙을 자주 수정해도 회귀를 빠르게 잡는다.

작업:
- Playwright 기반 E2E:
  - 이력서 확정 -> 채용공고 확정 -> 결과 생성
  - 이력서 수정 시 회사/결과 무효화
  - 회사 수정 시 결과 stale 처리
- SSE 로그 표시 스모크 테스트
- 실패 응답/빈 응답/스키마 오류에 대한 API 테스트 추가
- fixture 기반 golden test 도입 검토

완료 기준:
- 핵심 플로우 회귀가 CI 수준에서 바로 잡힌다.

### P5. 입력 확장과 배포 검토

목표:
- MVP 바깥의 입력 형식과 배포 제약을 다음 단계에서 검토한다.

작업:
- `pdf-parse` 기반 PDF 텍스트 추출 프로토타입
- 로컬 인증 의존성을 제거할 수 있는 구조 조사
- 원격 배포 시 인증 전략과 비밀 관리 방식 문서화

완료 기준:
- PDF 지원 feasibility가 확인된다.
- 로컬 전용 구조를 유지할지, 배포형으로 갈지 판단 근거가 생긴다.

## 4. 바로 다음 스프린트 추천

다음 1회 작업은 아래 범위가 적절하다.

- `generate-intro` 스키마에 `fitReasons[]` 추가
- 결과 페이지에 `매칭 근거` 카드 추가
- `company-to-json` fixture 기반 테스트 추가

이 조합이 가장 작고, 동시에 사용자 체감 개선폭이 크다.

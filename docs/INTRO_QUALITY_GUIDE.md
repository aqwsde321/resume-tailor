# 자기소개 품질 가이드

## 1) 목적

`generate-intro` 결과(`oneLineIntro`, `shortIntro`, `longIntro`)의 일관성과 신뢰도를 높이기 위한 기준 문서.

## 2) 품질 원칙

1. 사실 기반
- `resume.json`에 없는 경력/기술/성과를 생성하지 않는다.

2. 직무 정합성
- `company.json.requirements`와 `resume.json.techStack/experience`의 교집합을 우선 반영한다.

3. 간결성
- `oneLineIntro`: 25~45자 권장
- `shortIntro`: 120~220자, 2~4문장 권장
- `longIntro`: 450~700자, 5~8문장 권장

4. 톤
- 전문적, 담백한 톤
- 과장/추상 표현 최소화

## 3) 구조 템플릿

`oneLineIntro`
- 직무 + 핵심 역량 + 연차(또는 강점)

`shortIntro`
1. 직무/연차 요약
2. 핵심 프로젝트/성과 연결
3. 요구사항-경험 매핑 문장
4. (선택) 입사 후 기여 의지

`longIntro`
1. 직무/연차/핵심 강점 요약
2. 대표 프로젝트와 성과 설명
3. 협업 방식 또는 문제 해결 경험 연결
4. 공고 요구사항과 맞닿는 근거 구체화
5. (선택) 입사 후 기여 방향

## 4) 금지 규칙

- 검증 불가능한 수치 성과 임의 추가
- 회사명/직무명 오기
- 동일 의미 반복 문장 2회 이상
- 지나친 수식어(예: "압도적", "완벽한")

## 5) 빠른 품질 체크리스트

- [ ] 회사명/직무명이 `company.json`과 일치하는가?
- [ ] 핵심 기술이 `requirements`와 연결되는가?
- [ ] 경력 연차가 `resume.json.careerYears`와 맞는가?
- [ ] 과장/허위 문장이 없는가?
- [ ] 문장 길이와 읽기 흐름이 자연스러운가?

## 6) 개선 루프

1. 사용자 피드백 수집
- 어색한 표현, 과장 여부, 직무 핏 부족 사례

2. 실패 케이스 분류
- 사실 오류
- 직무 정합성 부족
- 문체/길이 문제

3. SKILL 가이드/프롬프트 수정
- 실패 유형별 제약 문장 추가

4. 재검증
- 동일 입력으로 전/후 비교

## 7) 차기 개선 항목(v0.3 제안)

- 출력 스키마 확장
  - `matchPoints[]`: 요구사항-근거 매핑
  - `riskNotes[]`: 애매한 포인트 명시
  - `confidence`: 0~1 점수
- UI 옵션
  - 톤(보수적/적극적)
  - 길이(짧게/보통)
